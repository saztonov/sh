# Homework Portal

монорепозиторий с четырьмя приложениями и общим пакетом типов.

## Правила работы

1. Всегда отвечать на русском языке.
2. Не коммитить в git самостоятельно — управление git остаётся за пользователем.
3. В диалогах и планах не писать код, переменные и другие технические подробности — описывать только архитектуру проекта и структуру БД.
4. При необходимости изменений в схеме БД — создавать отдельную миграцию, не изменяя существующие (они могут быть уже выполнены).
5. Все миграции выполняет пользователь вручную через SQL-редактор.

## Структура монорепозитория

- **apps/api** — бэкенд на Fastify 5, порт 3000
- **apps/web** — фронтенд на React 18 + Vite 6 + Ant Design 5, порт 5173
- **apps/scraper** — парсер Google Classroom на Playwright, работает по cron
- **apps/bot** — Telegram-бот на grammY
- **packages/shared** — общие типы и константы (предметы, дни недели, месяцы)

## Стек технологий

- Язык: TypeScript (везде)
- Пакетный менеджер: npm workspaces
- БД: Supabase (PostgreSQL), авторизация через Supabase Auth
- Файловое хранилище: S3-совместимое (Cloud.ru / Cloudflare R2)
- Деплой: Docker Compose (API, Web/Nginx, Scraper/Playwright, Bot), опционально Caddy для HTTPS

## Структура БД

Все таблицы защищены Row Level Security. Миграции хранятся в `supabase/`.
- актуальная схема БД хранится  в sql\schema\sql_schema.json.

- **courses** — курсы из Google Classroom (название, предмет, ссылка, активность)
- **course_mappings** — привязка ключевых слов к предметам (26 записей-сидов)
- **assignments** — домашние задания (курс, заголовок, описание, автор, баллы, дедлайн, статус, выполнено)
- **attachments** — файлы заданий (имя, MIME, размер, ключ S3, ссылка)
- **schedule_slots** — расписание уроков (день недели, номер урока, время, предмет)
- **scrape_runs** — логи запусков парсера (время, статус, найдено/новых, ошибка)
- **telegram_users** — пользователи бота (telegram_id, username, авторизация)

## API-маршруты

- `/api/health` — проверка здоровья
- `/api/assignments` — задания (список, детали, переключение выполнения)
- `/api/courses` — курсы (список, обновление предмета)
- `/api/schedule` — расписание (всё / по дню)
- `/api/files/:id/download` — скачивание файлов через presigned URL
- `/api/scraper/run` — ручной запуск парсера

## Страницы веб-приложения

- Вход (Supabase Auth)
- Задания (список с фильтрами, детальный просмотр в Drawer)
- Расписание (по дням недели)
- Настройки (управление привязкой курсов к предметам)

## Команды бота

`/start`, `/today`, `/tomorrow`, `/week`, `/schedule`

## Скрипты

- `npm run dev` — запуск всех приложений в режиме разработки
- `npm run dev:api|web|scraper|bot` — запуск отдельного приложения
- `npm run build` — сборка всех приложений
- `npm run seed` — заполнение расписания из Excel-файла
