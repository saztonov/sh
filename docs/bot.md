# Telegram-бот портала

Бот предоставляет голосовой доступ к порталу через AI-агент: принимает сообщения на естественном языке и выполняет операции с расписанием, заданиями и репетиторами.

---

## Возможности

### Просмотр заданий
- Задания на сегодня, завтра, текущую неделю
- Фильтрация по предмету или дате
- Только невыполненные / только выполненные
- Детали задания с вложенными файлами

### Расписание
- Расписание уроков на текущую и любую другую неделю
- Задания, привязанные к урокам

### Репетиторы
- Просмотр списка репетиторов и их предметов
- Добавить / переименовать / удалить репетитора
- Создать занятие — разовое или регулярное
- Изменить время или длительность занятия
- Перенести одно конкретное занятие без изменения расписания
- Удалить занятие

### Отметка выполнения
- Отметить задание выполненным или снять отметку

### Файлы
- Получить ссылку для скачивания вложения задания

### Парсер
- Запустить парсинг из Google Classroom, Eljur или оба источника сразу

---

## Команды

| Команда | Описание |
|---------|----------|
| `/start` | Регистрация и авторизация |
| `/today` | Задания на сегодня |
| `/tomorrow` | Задания на завтра |
| `/week` | Задания на эту неделю |
| `/schedule` | Расписание на сегодня |
| `/reset` | Сбросить контекст диалога с AI-агентом |

Все остальные сообщения обрабатывает AI-агент.

## Примеры запросов к агенту

```
покажи задания на сегодня
какие задания на следующей неделе по математике?
отметь задание по физике выполненным
покажи расписание на следующую неделю

добавь репетитора Иванов, предмет математика
покажи список репетиторов
создай занятие с Ивановым в пятницу в 16:00, длительность 1.5 часа
перенеси занятие с Ивановым с пятницы 14 марта на субботу 15 марта в 17:00
измени время занятия с Ивановым на 17:30
удали репетитора Иванова

запусти парсер Google Classroom
запусти парсинг из обоих источников
```

> Перед удалением бот уточнит, что именно будет удалено, и попросит подтверждение.

## Как пройти авторизацию

1. Написать боту `/start`
2. Для admin-аккаунта (`TELEGRAM_ADMIN_ID`) — авторизация автоматическая
3. Для остальных пользователей — создаётся запись в `telegram_users` с `is_authorized=false`; admin должен вручную поставить `is_authorized=true` в Supabase

## Настройка AI-провайдера

Добавить в `.env` один из вариантов:

### Cerebras (GLM-4.7, бесплатно по умолчанию)
```env
AI_PROVIDER=cerebras
CEREBRAS_API_KEY=<ключ с cloud.cerebras.ai>
# AI_MODEL=glm-4-32b  # по умолчанию
```

### Google AI Studio (Gemini 2.0 Flash, бесплатный tier)
```env
AI_PROVIDER=google
GOOGLE_AI_API_KEY=<ключ с aistudio.google.com>
# AI_MODEL=gemini-2.0-flash  # по умолчанию
# AI_MODEL=gemini-2.5-flash  # дешевле при платном использовании
```

### Groq (Llama 3.3 70B, бесплатный tier)
```env
AI_PROVIDER=groq
GROQ_API_KEY=<ключ с console.groq.com>
# AI_MODEL=llama-3.3-70b-versatile  # по умолчанию
```

### OpenRouter (любая модель)
```env
AI_PROVIDER=openrouter
OPENROUTER_API_KEY=<ключ с openrouter.ai>
# AI_MODEL=google/gemini-2.0-flash-001  # по умолчанию
```

OpenRouter даёт доступ к сотням моделей через один ключ. Модель задаётся в формате `провайдер/модель`.
Примеры популярных моделей с поддержкой tool_use:

| Модель | `AI_MODEL` | Цена |
|--------|-----------|------|
| Gemini 2.0 Flash | `google/gemini-2.0-flash-001` | бесплатно (лимит) |
| Gemini 2.5 Flash | `google/gemini-2.5-flash-preview` | дёшево |
| GPT-4o mini | `openai/gpt-4o-mini` | дёшево |
| Claude 3.5 Haiku | `anthropic/claude-3-5-haiku` | дёшево |
| Llama 3.3 70B | `meta-llama/llama-3.3-70b-instruct` | бесплатно (лимит) |
| GLM-4.7 | `thudm/glm-4-32b` | бесплатно (лимит) |

Актуальный список и цены: https://openrouter.ai/models

Для смены провайдера на работающем сервере — изменить `.env` и перезапустить контейнер бота (пересборка не нужна):

```bash
docker compose up -d bot
```

## Запуск в режиме разработки

```bash
# Из корня монорепозитория:
npm run dev:bot
```

Убедитесь, что в `.env` заданы:

```env
SUPABASE_URL=...
SUPABASE_SERVICE_ROLE_KEY=...
TELEGRAM_BOT_TOKEN=...
TELEGRAM_ADMIN_ID=...
AI_PROVIDER=cerebras          # или google / groq
CEREBRAS_API_KEY=...          # ключ выбранного провайдера
```

## Деплой на сервере

Бот запускается в Docker-контейнере `sh-bot-1` — он автоматически стартует после healthcheck API.

Обновить бота после изменений в коде:

```bash
cd ~/www/sh
git pull
docker compose up -d --build bot
```

Только перезапустить (без пересборки, например после изменения `.env`):

```bash
cd ~/www/sh
docker compose restart bot
```

Посмотреть логи:

```bash
docker compose logs -f bot
docker compose logs --tail 100 bot
```

## Панель администратора (веб)

После выполнения миграции `sql/migrations/20260312_agent_logs.sql` в разделе **AI-агент** веб-интерфейса (только для роли `admin`) доступны:

- Статистика токенов за день / неделю / месяц / всё время
- Разбивка по провайдерам и пользователям
- Таблица всех событий с фильтрами
- Детальный просмотр сессии: полная цепочка запрос → tool_call → ответ

## Ограничения

- История диалога хранится в памяти (сбрасывается при перезапуске бота или через `/reset`)
- Контекст автоматически очищается после 30 минут неактивности
- В одном ответе агент выполняет максимум 8 последовательных шагов
